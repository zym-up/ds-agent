# 页面布局美化 设计文档

## 背景

当前 Streamlit 应用的页面结构存在以下问题：
- 5 个导航项过多，历史项目使用频率低却占主导航位
- 报告导出功能独立成页，与主工作流割裂
- 无法在已打开项目中追加新数据
- 分析对话页面交互不够精细

## 设计目标

1. 精简导航至 3 项，侧边栏增加 3 个折叠面板
2. 报告导出改为分析对话页内的弹窗
3. 支持项目内追加数据，多数据源勾选合并分析
4. 优化分析对话页交互细节

---

## 界面清单

| # | 界面 | 类型 | 说明 |
|---|------|------|------|
| 1 | 侧边栏 | 全局 | 3 导航 + 3 折叠面板 + 当前项目显示 |
| 2 | 新建项目页 | 页面 | 命名 + 上传数据 → 自动进入分析 |
| 3 | 分析对话页 | 页面 | 主工作区：工具栏 + 三栏布局 |
| 4 | 设置页 | 页面 | LLM API 配置 + 关于 |
| 5 | 上传新数据弹窗 | 弹窗 | 文件上传 → 加入数据列表 |
| 6 | 报告导出弹窗 | 弹窗 | 勾选步骤 + 补充观点 → 预览 |
| 7 | 项目信息弹窗 | 弹窗 | 名称/创建时间/数据量 |
| 8 | 报告预览区 | 页面 | AI 结论 + 完整报告预览 → 导出/下载 |

## 详细设计

### 1. 侧边栏（全局）

**导航区：**
- 🆕 新建项目
- 🔬 分析对话
- ⚙ 设置

**三个折叠面板（默认全部折叠，可同时展开）：**

#### 📂 数据列表
- 展开后显示当前项目所有数据文件
- checkbox 多选，默认全选
- 全选 / 取消全选快捷操作
- 每个文件显示：文件名 + 行数
- 底部显示「已选 N 个文件，合计 X 行」
- 底部「+ 上传新数据」链接 → 弹出上传弹窗
- 列表超出时滚动

#### 📁 历史项目
- 展开后显示所有历史项目列表
- 每项显示：项目名 + 创建日期
- 点击切换项目，当前项蓝色高亮
- 列表超出时滚动
- 空状态：「暂无项目」

#### 📋 历史报告
- 展开后显示当前项目的历史报告列表
- 每项显示：报告名 + 生成日期
- 点击在新标签页或预览区打开查看
- 列表超出时滚动
- 空状态：「暂无报告」

**当前项目信息：**
- 分隔线下方灰色小字显示「当前项目: xxx」
- 无项目时不显示

### 2. 新建项目页

**路由：** 侧边栏点击「🆕 新建项目」

**内容：**
- 标题「🆕 新建项目」+ 当前项目名（如有）
- 项目名称输入框
- 数据文件上传区（拖拽或点击，支持 CSV/Excel/JSON）
- 「创建并开始分析」按钮
- 约束：必须上传数据才能创建项目
- 创建成功后自动跳转到分析对话页

### 3. 分析对话页（主界面）

#### 顶部工具栏
- 页面标题「🔬 分析对话」
- 项目名（蓝色标签，可点击 → 弹出项目信息弹窗）
- 数据概况：分析数据 X 个文件 · XX,XXX 行
- 📤 上传新数据 按钮 → 弹窗
- 📋 导出报告 按钮 → 弹窗

#### 三栏布局（25% / 50% / 25%）

**左栏：💬 对话**
- 对话消息列表（👤 用户 + 🤖 Agent 角色区分）
- 用户控制滚动条（不自动滚动）
- 底部 chat_input 输入框
- 追问时自动带已完成步骤摘要作为上下文

**中栏：📈 结果**
- 顶部显示当前查看的步骤名称
- AI 解读：蓝色左边框引用块（🤖 AI 解读: ...）
- 图表面板（Plotly 图表）
- 数据表格（可折叠）
- 查看不同步骤时切换显示内容

**右栏：📋 分析步骤**
- 步骤列表，每项显示：状态图标 + 编号 + 描述
- 状态：○ 待执行 / ⟳ 执行中 / ✓ 已完成 / ✗ 错误
- 已完成步骤可点击查看结果 → 中栏切换 + 该步骤高亮
- 待执行步骤旁有 ▶ 执行按钮
- 全部完成时显示「所有步骤已完成」

#### 交互逻辑
- 点击已完成步骤 → 高亮该步骤 + 结果区切换显示对应结果
- 上传新数据后 → 加入数据列表 → 提示「数据已追加，是否重新分析？」
- 数据列表勾选变化 → 自动重新合并数据 → 提示用户

### 4. 设置页

**Tab 1: LLM API 配置**
- 预设模板选择（DeepSeek / Qwen / 自定义）
- base_url / api_key / model / temperature / max_tokens 输入
- 测试连接 + 保存配置按钮

**Tab 2: 关于**
- 功能介绍文字（保持不变）

### 5. 上传新数据弹窗

**触发：** 分析对话工具栏「📤 上传新数据」或侧边栏数据列表底部「+ 上传新数据」

**内容：**
- 标题「📤 向当前项目追加数据」
- 提示文字：新数据将与 {project_name} 已有数据按列名匹配、按行拼接
- 文件上传区（拖拽或点击）
- 列名匹配预览（如有不一致给出警告）
- 取消 / 确认追加 按钮
- 确认后：文件加入数据列表 → 默认勾选 → 提示是否重新分析

### 6. 报告导出弹窗

**触发：** 分析对话工具栏「📋 导出报告」

**内容：**
- 标题「📋 导出分析报告」
- 报告标题输入框（默认值：{project_name} — 分析报告）
- 包含内容：勾选列表（已完成步骤 + AI 综合结论），默认全选
- 工程师补充观点输入框（可选）
- 三个按钮：取消 / 🤖 生成 AI 结论并预览 / ⬇ 直接导出 HTML

**按钮行为：**
- 取消 → 关闭弹窗
- 生成 AI 结论并预览 → 弹窗关闭 → 跳转到结果区显示 AI 综合结论预览（调用 summarize_conclusions）
- 直接导出 HTML → 生成 HTML 文件 → 下载

### 7. 项目信息弹窗

**触发：** 点击分析对话工具栏中的项目名

**内容：**
- 项目名称
- 创建时间
- 数据文件数量 + 总行数
- 分析步骤数
- 报告数量

### 8. 报告预览区

**位置：** 分析对话页的结果区域（替换当前步骤结果显示）

**内容：**
- 完整报告预览（HTML 嵌入）
- 返回分析 / 下载 HTML 按钮

## 数据模型变更

### ProjectManager 需支持

- `add_data(project_id, file_path)` — 向项目追加数据文件，按列名匹配拼接
- `list_data_files(project_id)` — 列出项目所有数据文件
- `get_data_summary(project_id)` — 获取数据概况（文件数、总行数）
- `merge_selected_data(project_id, selected_files)` — 按勾选文件合并数据
- `list_reports(project_id)` — 列出项目所有报告
- `get_report(project_id, report_name)` — 获取报告内容
- `get_project_info(project_id)` — 获取项目元信息

### 数据文件存储
- `projects/{project_id}/data/` 下存储多个文件：data_1.csv, data_2.csv, ...
- 文件索引存储于 meta.json 或独立的 data_manifest.json

## 空状态处理

| 场景 | 表现 |
|------|------|
| 无任何项目 | 历史项目面板显示「暂无项目」；分析对话页提示「请先新建项目」 |
| 项目无数据文件 | 数据列表面板显示「暂无数据，请上传」 |
| 无数据选中 | 全选取消时，分析对话页提示「请至少选择一个数据文件」 |
| 无分析步骤 | 对话区提示「请描述你的分析需求」 |
| 无历史报告 | 历史报告面板显示「暂无报告」 |
